# Documentation

Ce dossier centralise la documentation structuree du projet.

## Arborescence

- `planning/` : backlog, plan d'action, plan de refonte, rituels de nettoyage
- `reviews/` : revues de code et syntheses de revues
- `ui/` : documentation active UI (gouvernance, qualite, rapports, release)
- `archive/` : historiques de phases/revues

Point d'entree UI:
- `ui/README.md`

Les anciens chemins de documents conserves a la racine restent disponibles
via des fichiers ponts pour compatibilite des liens historiques.
